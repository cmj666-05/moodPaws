# 开源组件与依赖清单

## 结论

当前仓库不是直接 fork 的完整开源业务项目，而是基于多个开源框架和组件组合开发。

| 子项目 | 说明 |
| --- | --- |
| `app/` | Vue 3 + Vite + Echarts+Capacitor 主应用 |
| `moodpaws-server/` | Node.js + Express |
| `web/` | React + TypeScript + Vite |

## app

| 组件 | 作用 |
| --- | --- |
| `vue` | 前端框架 |
| `vite` | 开发与构建工具 |
| `@vitejs/plugin-vue` | Vue SFC 编译支持 |
| `@capacitor/core` | Capacitor 运行时核心 |
| `@capacitor/cli` | Capacitor 命令行工具 |
| `@capacitor/android` | Android 平台支持 |
| `echarts` | 图表可视化 |
| `dayjs` | 日期时间处理 |

| 模板痕迹 | 说明 |
| --- | --- |
| `app/README.md` | 仍保留默认 `Vue 3 + Vite` 模板内容，说明该目录最初基于标准脚手架创建 |

## moodpaws-server

| 组件 | 作用 |
| --- | --- |
| `express` | HTTP 服务框架 |
| `mqtt` | MQTT 客户端 |
| `bonjour-service` | mDNS / Bonjour 局域网服务发现 |
| `cors` | 跨域支持 |
| `dotenv` | `.env` 配置加载 |

| 来源特征 | 说明 |
| --- | --- |
| 服务端结构 | 更像基于常见 Node 开源组件自行搭建的业务服务 |

## web

| 组件 | 作用 |
| --- | --- |
| `react` | 前端框架 |
| `react-dom` | React DOM 渲染 |
| `react-router-dom` | 前端路由 |
| `typescript` | 类型系统 |
| `vite` | 开发与构建工具 |
| `@vitejs/plugin-react` | React 编译支持 |
| `eslint` | 代码检查 |
| `@typescript-eslint/eslint-plugin` | TypeScript ESLint 规则 |
| `@typescript-eslint/parser` | TypeScript ESLint 解析器 |
| `eslint-plugin-react-hooks` | Hooks 规则检查 |
| `eslint-plugin-react-refresh` | React Refresh 规则支持 |
| `@types/react` | React 类型声明 |
| `@types/react-dom` | React DOM 类型声明 |

| 模板痕迹 | 说明 |
| --- | --- |
| `web/` | 明显是标准 `React + TypeScript + Vite` 脚手架衍生项目 |

## 补充说明

| 项目判断 | 说明 |
| --- | --- |
| 仓库性质 | 不应把整个仓库视为单一开源项目的直接改造版 |
| `web/` 定位 | 不应把 `web/` 误认为当前主 App 的核心前端 |
| 更准确的描述 | 仓库使用了多个成熟开源框架与组件，但业务实现主要为自研组合 |
