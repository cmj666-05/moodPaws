# 局域网视频接入说明

本文档说明如何在不修改 Python 图传脚本的前提下，把 MoodPaws App 接到局域网视频流。

## 当前结构

- 主后端：`moodpaws-server`
- 视频服务：独立的 Python Flask 图传服务
- App：先发现主后端，再从主后端拿视频配置

Python 图传脚本当前暴露的接口为：

- 视频流：`http://<视频设备地址>:5000/video_feed`
- 健康检查：`http://<视频设备地址>:5000/health`

## 推荐接法

推荐让主后端发布视频服务地址，App 不直接写死视频设备 IP。

优点：

- 主后端和 App 的发现链路保持统一
- 视频设备 IP 变化时，不需要重新打包 App
- 不需要修改 Python 图传脚本

## 主后端配置方式

在 `moodpaws-server/.env` 中配置以下字段，三种方式任选一种：

### 方式 1：直接配置完整地址

```env
VIDEO_STREAM_ENABLED=true
VIDEO_STREAM_URL=http://camera-box.local:5000/video_feed
```

### 方式 2：配置源站地址

```env
VIDEO_STREAM_ENABLED=true
VIDEO_STREAM_ORIGIN=http://camera-box.local:5000
VIDEO_STREAM_PATH=/video_feed
```

### 方式 3：配置主机名或 `.local` 名称

```env
VIDEO_STREAM_ENABLED=true
VIDEO_STREAM_HOST=camera-box.local
VIDEO_STREAM_PORT=5000
VIDEO_STREAM_PATH=/video_feed
```

## 推荐优先级

推荐优先使用以下顺序：

1. `VIDEO_STREAM_URL`
2. `VIDEO_STREAM_ORIGIN`
3. `VIDEO_STREAM_HOST + VIDEO_STREAM_PORT + VIDEO_STREAM_PATH`

如果视频设备支持 `.local` 名称，优先使用 `.local`，因为它比手填 IP 更适合“局域网 IP 会变化”的场景。

## App 当前行为

App 当前的视频地址解析顺序为：

1. `VITE_VIDEO_STREAM_URL`
2. 主后端 `/api/health` 返回的 `video.url`
3. 主后端 `/api/health` 返回的 `video.origin`
4. 主后端 `/api/health` 返回的 `video.host + port + path`
5. 独立 mDNS 视频发现

因此，只要主后端配置好了视频地址，App 就不需要单独知道 Python 图传服务的 IP。

## 你现在最推荐的配置

如果你的 Python 视频设备能在局域网里被 `.local` 名称访问，建议直接这样填：

```env
VIDEO_STREAM_ENABLED=true
VIDEO_STREAM_HOST=<你的视频设备名>.local
VIDEO_STREAM_PORT=5000
VIDEO_STREAM_PATH=/video_feed
```

如果你已经知道完整可访问地址，则直接填：

```env
VIDEO_STREAM_ENABLED=true
VIDEO_STREAM_URL=http://<你的视频设备名或地址>:5000/video_feed
```
